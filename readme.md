## Реализация Transformer на PyTorch

Этот код реализует архитектуру Transformer (энкодер-декодер) с нуля на PyTorch и обучает её задаче seq2seq – обращению строк. Пример разработан для демонстрации ключевых компонентов: позиционного кодирования, многоголового внимания, масок, teacher forcing и жадного декодирования. Комментарии ориентированы на студентов и инженеров, изучающих Transformer.

---

## 1. Задача

Модель решает задачу **обращения строк**:

*   **Вход:** строка символов (например, `"hello"`)
*   **Выход:** перевернутая строка ("olleh")

Это хороший учебный пример, поскольку:

*   Задача понятна и не требует сложной предобработки данных.
*   Последовательности короткие (до 12 символов), что упрощает отладку.
*   Задача требует установления зависимостей между позициями символов (буква в позиции `i` входа должна повлиять на позицию `n-i-1` выхода).
*   Позволяет наглядно проверить работу механизма внимания и масок.

Архитектура Transformer реализована полностью самостоятельно (без использования `nn.Transformer`), что обеспечивает полное понимание всех деталей.

---

## 2. Позиционное кодирование (`PositionalEncoding`)

```python
class PositionalEncoding(nn.Module):
```

**Зачем:**

Transformer не использует рекуррентные или сверточные слои, поэтому информация о порядке токенов должна быть добавлена явно. Позиционное кодирование добавляет к эмбеддингу каждого токена вектор, зависящий от его позиции в последовательности.

**Как работает:**

*   Создается матрица `pe` размером `[max_len, d_model]`.
*   Для четных индексов (`0::2`) используется синус, для нечетных – косинус.
*   Формула из оригинальной статьи "Attention is All You Need":
    *   `PE(pos, 2i) = sin(pos / 10000^(2i/d_model))`
    *   `PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))`

**Важные моменты:**

*   `register_buffer` сохраняет тензор как часть модуля, но не как обучаемый параметр (он не обновляется в процессе обучения).
*   В методе `forward` происходит сложение `x + self.pe[:, :x.size(1), :]` – учитывается, что длина входной последовательности может быть меньше `max_len`.
*   После сложения применяется `Dropout` для регуляризации.

## 3. Многоголовое внимание (`MultiHeadAttention`)

Это **ключевая часть** Transformer. Код реализует полное скалярное внимание (scaled dot-product attention) с разделением на несколько "голов".

**Основные шаги:**

1.  **Линейные проекции:**  `Q`, `K`, `V` преобразуются из `d_model` в `d_model`.
2.  **Разделение на головы:**  тензор `[B, L, D]` преобразуется в `[B, H, L, D_k]`, где `D_k = D / H`.  `B` - размер батча, `L` - длина последовательности, `D` - размерность вектора.
3.  **Вычисление scores:** `Q * K^T / sqrt(D_k)` – вычисляется "внимание" между всеми парами позиций.
4.  **Применение масок:**
    *   `attn_mask` – маска для предотвращения доступа к будущим позициям (look-ahead) или произвольная матрица маскирования.
    *   `key_padding_mask` – маска для игнорирования паддингов (заполняющих токенов) в ключах/значениях.
5.  **Softmax** применяется к последней оси, затем `Dropout`.
6.  **Умножение на V:**  результат умножается на матрицу `V`, и все головы объединяются обратно в `[B, L_q, D]`.
7.  **Финальная проекция:**  `W_o` – линейный слой для финального преобразования.

**Важные моменты:**

*   Внимание реализовано вручную с использованием `matmul` для наглядности и удобства отладки.
*   Поддерживаются маски разных размерностей (2D/3D/4D) для гибкости.
*   `masked_fill` с `float('-inf')` гарантирует, что замаскированные позиции получат нулевой вес после применения softmax.

**Для студентов:**

`attn_mask` и `key_padding_mask` могут использоваться одновременно. Например, в декодере нужно запретить доступ к будущим токенам (`attn_mask`) и одновременно игнорировать паддинги (`key_padding_mask`).

## 4. Слой энкодера (`TransformerEncoderLayer`)

Слой энкодера состоит из двух подслоев, каждый из которых содержит:

*   **Pre-LayerNorm:** Нормализация слоя (Layer Normalization) применяется перед остаточным соединением.  В коде порядок: внимание → нормализация → добавление, затем FFN → нормализация → добавление.
    *   В оригинальной статье часто используется Post-LayerNorm, но Pre-Norm обеспечивает более стабильное обучение.

**Внимание – самовнимание (self-attention):**

`query = key = value = src`. Это позволяет каждому токену взаимодействовать со всеми другими токенами в той же последовательности.

**Feed-forward network:**

Два линейных слоя с функцией активации ReLU между ними (`d_model → dim_feedforward → d_model`).  Применяется к каждой позиции независимо.

**Важный момент:**

В энкодере `src_mask` обычно равен `None` (нет запрета на доступ к будущим позициям, так как энкодер видит всю входную последовательность сразу).  Однако, при решении задач типа языкового моделирования маска может понадобиться.

---

## 5. Слой декодера (`TransformerDecoderLayer`)

Декодер сложнее и включает **три подслоя**:

1.  **Self-attention (замаскированное)** – декодер не должен видеть будущие токены целевой последовательности. Используется `tgt_mask` (look-ahead).
2.  **Cross-attention** – декодер "смотрит" на выход энкодера (`memory`). Здесь `query` берется из декодера, а `key` и `value` – из памяти энкодера. Это позволяет связать целевые токены с исходными.
3.  **Feed-forward** – как в энкодере.

**Маски:**

*   `tgt_mask` – запрещает просмотр вперед при self-attention.
*   `memory_mask` – позволяет запретить доступ к отдельным позициям в памяти (редко используется).
*   `tgt_key_padding_mask` – для паддингов в самом декодере.
*   `memory_key_padding_mask` – для паддингов во входной последовательности (чтобы декодер не обращал внимания на заполняющие токены).

**Важный момент:**

В cross-attention не нужна маска look-ahead, поскольку память энкодера уже полностью доступна. Однако необходимо использовать `memory_key_padding_mask`, чтобы декодер не учитывал пустые токены во входной последовательности.

---

## 6. Стеки энкодера и декодера (`TransformerEncoder`, `TransformerDecoder`)

Эти классы просто повторяют переданный слой `num_layers` раз, последовательно применяя его к выходу предыдущего слоя. Используется `nn.ModuleList` для правильной регистрации подмодулей.

---

## 7. Полная модель (`Transformer`)

Объединяет все компоненты:

*   **Эмбеддинг (`nn.Embedding`)** – преобразует индексы токенов в векторы размерности `d_model`.
*   **Масштабирование эмбеддингов** – `* math.sqrt(self.d_model)`.  Это важный практический прием, который сохраняет дисперсию после сложения с позиционным кодированием.
*   **Позиционное кодирование** – добавляет информацию о порядке токенов.
*   **Энкодер** – принимает `src`, возвращает `memory`.
*   **Декодер** – принимает `tgt` и `memory`, возвращает признаки.
*   **Выходной линейный слой** – проецирует `d_model` обратно в `vocab_size` для получения логитов по каждому токену.

Методы `encode` и `decode` вынесены для инференса – позволяют закэшировать выход энкодера и пошагово генерировать ответ.

---

## 8. Токенизатор (`CharTokenizer`)

Поскольку задача – обращение строк на уровне символов, токенизатор работает с отдельными символами (буквы и пробел).  Добавляются специальные токены:

*   `<pad>` – для выравнивания последовательностей в батче.
*   `<sos>` / `<s>` – маркеры начала и конца последовательности.
*   `<unk>` – для неизвестных символов (в данном коде не используется, но добавлен для полноты).

Методы `encode` и `decode` автоматически добавляют/удаляют `<sos>` и `</s>`.

**Важный момент:**

В задаче обращения строки `</s>` нужен, чтобы декодер знал, когда остановиться.  В сгенерированной последовательности он оставляется и удаляется при декодировании.

---

## 9. Датасет (`ReverseStringDataset`)

Генерирует случайные строки длины от `min_len` до `max_len` из букв и пробела.  Для каждой пары `(src, tgt)` сразу же токенизирует с добавлением `<sos>` и `</s>`.

**Важный учебный момент:**

Эти последовательности **разной длины**. PyTorch требует, чтобы тензоры в батче имели одинаковую длину, поэтому используется `collate_fn` с `pad_sequence`, который дополняет все последовательности до максимальной длины в батче значением `pad_idx`.

---

## 10. Функции для масок и декодирования

*   `generate_square_subsequent_mask(sz)` – создает верхнюю треугольную матрицу из `-inf` над главной диагональю. При сложении с `scores` эти позиции обнуляются после применения softmax – декодер не видит будущие токены.
*   `greedy_decode` – пошаговая генерация:
    *   На каждом шаге подаем в декодер уже сгенерированную последовательность (начинающуюся с `<sos>`), получаем логиты для последнего токена, выбираем индекс с максимальной вероятностью, добавляем к выходу.
    *   Процесс останавливается при появлении `</s>` или достижении `max_len`.

**Важный момент:**

На инференсе `encode` вызывается один раз (получаем `memory`), а затем циклически вызывается `decode` с растущим `tgt`. Это эффективно, поскольку энкодер не пересчитывается.

## 11. Обучение

### Параметры

*   `BATCH_SIZE = 64`, `EPOCHS = 15`, `LR = 0.001`.
*   `D_MODEL = 128`, `NHEAD = 4`, `NUM_LAYERS = 3`. Уменьшенные значения для быстрого обучения на CPU.

### Подготовка данных для teacher forcing

```python
tgt_input = tgt[:, :-1]   # все токены, кроме последнего (</s>)
tgt_output = tgt[:, 1:]   # все токены, кроме первого (<sos>)
```

На каждом шаге модель получает правильный предыдущий токен, что ускоряет обучение.

### Маски

*   `tgt_mask` – look-ahead для декодера (размер `[tgt_len, tgt_len]`).
*   `src_key_padding_mask` – указывает, где в исходной последовательности стоят паддинги (`True` = это padding).
*   `tgt_key_padding_mask` – аналогично для `tgt_input`.
*   `memory_key_padding_mask` = `src_key_padding_mask` (передается в декодер, чтобы при cross-attention не учитывать паддинги энкодера).

### Функция потерь

`CrossEntropyLoss(ignore_index=tokenizer.pad_idx)` – токены паддинга не участвуют в расчете потерь.

### Градиентное клиппирование

`clip_grad_norm_(model.parameters(), 1.0)` – предотвращает взрыв градиентов.

### Валидация после каждой эпохи

Выводится один пример из валидационного датасета с предсказанием, полученным с помощью жадного декодирования.

## 13. Ключевые выводы

1.  **Transformer – это не "черный ящик"**: его компоненты (внимание, нормализация, остаточные связи) легко реализовать вручную, и этот код дает полное представление об их работе.
2.  **Маски играют критическую роль**: без look-ahead маски декодер бы жульничал и просто копировал вход, а без маски паддингов модель бы училась на мусорных позициях.
3.  **Разделение энкодера и декодера** позволяет гибко использовать модель на этапе инференса.
4.  **Teacher forcing** ускоряет обучение, но на инференсе переключаемся на авторегрессивный режим (жадный или более сложный).
5.  **Простая задача (обращение строк)** – идеальный полигон для проверки корректности внимания: успех показывает, что модель способна устанавливать зависимости между удаленными позициями.
